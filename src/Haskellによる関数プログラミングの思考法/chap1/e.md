数値上の加法は結合性を持つ

3つの数値をa,b,cで表すと
(a + b) + c 
= a + b + c 
= a + (b + c)

のため

リスト上の連結も結合性を持つ
3つのリストを[a], [b, c], [d, e, f] とすると
([a] ++ [b, c]) ++ [d,e,f]
= [a] ++ [b, c] ++ [d, e, f]
= [a] ++ ([b,c] ++ [d,e,f])
のため

関数合成も結合性を持つ
3つの関数をf : c -> d, g : b -> c, h : a -> b とすると
(f . g) . (h)
= ((c -> d) . (b -> c)). (h)
= (b -> d). (h)
= (b -> d) . (a -> b)
= a -> d
また、
f . (g . h)
= f . ((b -> c) . (a -> b))
= f . (a -> c)
= (c -> d) . (a -> c) 
= a -> d 
のため


数値上の演算で、結合性を持たないものは、割り算
ex. a = 10, b = 2, c = 5とすると
(a / b) / cは
(10 / 2) / 5 = 1 
a / (b / c)は
10 / (2 / 5) 
= 10 / 0.4 
= 25

よって、結合性を持たない

加法の単位元は0
ある数をaとすると
a + 0 = 0 + a = a が成り立つため

連接の単位元は[]
あるリストを[a,b,c]とすると
[a,b,c] ++ [] = [] ++ [a,b,c] = [a,b,c]
のため

関数合成の単位元はid 
ある関数をfとすると
f . id = id . f = f のため
